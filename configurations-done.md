## Encountered issues

- docker not properly starting
    solution
    - stop docker service
        ``` sudo docker compose -f docker-compose.dev.yaml down -v```
    - restart docker 
        ``` sudo docker compose -f docker-compose.dev.yaml up -d```
    - check temporal status
        ``` sudo docker compose -f docker-compose.dev.yaml logs -f temporal```
        ``` sudo docker compose -f docker-compose.dev.yaml exec temporal tctl --address temporal:7233 namespace list ```
    - if temporal is not running, restart it
        ``` sudo docker compose -f docker-compose.dev.yaml restart temporal```


- Limit to number of watchers
    solution
    - sudo sysctl fs.inotify.max_user_watches=524288    #temporary increase the number of watchers
    - echo "fs.inotify.max_user_watches=524288" | sudo tee /etc/sysctl.d/99-watchers.conf  #permanent increase the number of watchersq
    - sudo sysctl --system  #apply the changes


- temporal did not start as expected
    solution
    - sudo docker compose -f docker-compose.dev.yaml logs -f temporal   #check the logs
    - added health check to temporal postgres...
    - restarted thee docker service
